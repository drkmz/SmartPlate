## What It Does

SmartPlate is a full-stack web app that helps users eat smarter. Users create an account, set dietary preferences, and get access to:

- **PlateBot** — an AI nutrition assistant powered by Claude Sonnet that answers food questions and gives personalized advice
- **AI Meal Plans** — a daily 4-meal plan generated by Claude based on each user's calorie goals and dietary restrictions
- **Recipe Generator** — search by ingredient and browse recipes pulled from TheMealDB
- **Nutrition Tracker** — log meals and track daily calories, protein, carbs, and fat; nutrition data fetched live from the USDA FoodData Central API
- **Nutrition Explorer** — browse the full USDA food database
- **Shopping List** — AI-generated grocery list from saved recipes, organized by category
- **Favorites** — save recipes to a personal collection

## My Role — Project Lead & AI Engineer

I led the project and owned all AI/API integration:

- **PlateBot AI chatbot** — designed the full conversation flow, system prompt engineering, and Claude API integration (`includes/ai-helper.php`). PlateBot is context-aware: it pulls each user's dietary restrictions, calorie goals, and macro targets from the database and injects them into the system prompt so responses are personalized.
- **AI Meal Plan generation** — built the endpoint (`PHP/get_meal_plan.php`) that prompts Claude to generate a structured JSON meal plan tailored to the user's preferences, then parses and persists it to the database.
- **AI Shopping List** — Claude generates a categorized grocery list from the user's saved recipes (`PHP/shopping-list.php`).
- **USDA FoodData Central integration** — nutrition data auto-fetch when logging meals (`PHP/log_meal.php`).
- **TheMealDB integration** — recipe search and detail pages (`PHP/api_recipe_generator.php`, `PHP/recipe_detail.php`).
- **Frontend** — landing page, login/signup UI, and navigation.
- **Deployment** — configured and deployed the app to Hostinger shared hosting.

## Technical Highlights

- **Prompt engineering** — system prompts are dynamically constructed per-user by pulling their profile from MySQL and building a context string before each Claude API call, keeping responses relevant without exposing raw data to the client
- **PDO prepared statements** throughout — no raw SQL string interpolation, protecting against SQL injection
- **Session-based auth** with PHP's native `password_hash` (bcrypt) — no third-party auth libraries
- **AJAX-driven UI** — meal logging, chat, and meal plan generation all use fetch/XHR so the page never fully reloads
- **Graceful API fallback** — if USDA returns no nutrition data for a logged meal, the entry still saves with zero values rather than erroring out

## Running Locally

**Prerequisites:** PHP 8.x, MySQL, a local server (XAMPP / MAMP / Laragon)

```bash
git clone https://github.com/drkmz/SmartPlate.git
cd SmartPlate
```

1. Create a MySQL database and import `database/smart_plate_db.sql`
2. Fill in `config/config.php` with your DB credentials
3. Create `config/api-keys.php` with your API keys:

   ```php
   <?php
   define('AI_API_KEY',  'your-claude-api-key');
   define('AI_MODEL',    'claude-sonnet-4-6');
   define('FDC_API_KEY', 'your-usda-fdc-api-key');
   ```

4. Point your local server's root at the project folder and visit `localhost`
